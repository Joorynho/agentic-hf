## 1. Overview

MVP2 completes the full 5-pod firm: four new strategy pods (Beta/Gamma/Delta/Epsilon), LLM-powered CEO/CIO governance with graceful fallback, a full CRO enforcement layer, Polymarket CLOB integration, a capital allocation engine, agent collaboration loops, and four new TUI screens including the animated building view.

### Approach
Sequential layers — infrastructure → pods → governance agents → TUI. Each layer is independently testable before the next begins.

### Key Decisions
| Decision | Choice | Rationale |
|---|---|---|
| LLM fallback | Graceful stub mode when no API key | System always runs; LLM enhances |
| Pod agent depth | Full 6-agent structure with real strategy logic | MVP2 is the full firm |
| Polymarket | Full CLOB integration (orderbook depth + implied prob) | Richest signal |
| Collaboration | Bounded loops (max 5–10 iter) with transcript audit | Deliberation not just data passing |
| Cross-pod isolation | Preserved — CEO/CIO/CRO use governance channel only | Pod isolation is non-negotiable |

## 3. Polymarket CLOB Adapter

### `src/data/adapters/polymarket_adapter.py`
- Fetches markets from `https://clob.polymarket.com/markets` filtered by tags
- Fetches orderbook from `https://clob.polymarket.com/orderbook/{token_id}` for best bid/ask
- Computes `implied_prob = yes_price / (yes_price + no_price)`
- Caches to Parquet via existing `ParquetCache` layer
- Graceful fallback: API key missing or request fails → returns empty list, logs INFO
- Rate limiting: `tenacity` retry with exponential backoff
- Pods Gamma, Delta, Epsilon receive `PolymarketSignal` via their `PodGateway.subscribe_polymarket()`

### API Key
`POLYMARKET_API_KEY` loaded from `.env` via `python-dotenv`. Never hardcoded.

## 4. Capital Allocation Engine

### `src/backtest/accounting/capital_allocator.py`

```python
class CapitalAllocator:
    def current_allocations(self) -> dict[str, float]: ...
    def propose_rebalance(self, summaries: list[PodSummary]) -> list[AllocationRecord]: ...
    def apply_allocation(self, records: list[AllocationRecord]) -> None: ...
    def validate(self, records: list[AllocationRecord]) -> bool: ...
    # Invariant: sum(allocations.values()) == 1.0, all >= 0
```

- Every `apply_allocation()` call writes `AllocationRecord` list to DuckDB audit log
- Publishes `ALLOCATION_CHANGE` event on bus
- Validates: sum = 1.0, no pod < 0%, no single pod > 50% without CRO sign-off

## 5. Agent Collaboration Loops

### Loop Types and Iteration Limits

| Loop | Participants | Max Iter | Trigger |
|---|---|---|---|
| 1. Researcher → Signal refinement | Researcher ↔ Signal | 5 | Every signal generation cycle |
| 2. Signal → PM challenge | Signal ↔ PM | 5 | Before every trade decision |
| 3. PM → Risk sign-off | PM ↔ Pod Risk | 10 | Before every order submission |
| 4. CIO ↔ Pod PM negotiation | CIO ↔ Pod PM (via gateway) | 5 | Capital rebalance events |
| 5. CRO ↔ Pod Risk interrogation | CRO ↔ Pod Risk Agents | 5 | Risk breach detection |
| 6. CEO ↔ CIO ↔ CRO deliberation | CEO ↔ CIO ↔ CRO | 5 | Mandate updates |
| 7. CEO ↔ CIO strategy co-decision | CEO ↔ CIO (+ CRO join) | 10 | Strategy regime changes |

### Deadlock / No-Consensus Handling
If max iterations reached without consensus:
- Intra-pod (Loops 1–3): conservative default (no trade, hold, or escalate to CRO)
- Governance (Loops 4–7): hold current allocation, publish `FIRM_ALERT` to bus, require human acknowledgement in TUI F6

### Isolation Preserved in Governance Loops
- CIO/CEO/CRO communicate with pod agents via `governance.{pod_id}` topic only
- Pod agents respond with sanitised `PodSummary` + structured Q&A messages
- Pod agents never expose raw positions, signals, or model params across the boundary

### `src/core/bus/collaboration_runner.py`
```python
class CollaborationRunner:
    async def run_loop(
        self,
        topic: str,
        participants: list[BaseAgent],
        max_iterations: int,
        initial_message: AgentMessage,
    ) -> CollaborationLoop: ...
```

All loop transcripts written to DuckDB audit log. `CollaborationLoop` published as `COLLABORATION_END` event when complete.

## 6. Firm-Level Governance Agents

### CEO Agent (`src/agents/ceo/ceo_agent.py`)
- **LLM mode:** Claude receives all `PodSummary` objects + current market regime + CIO input
- **Tool:** `approve_mandate(narrative, objectives, constraints, rationale)`
- **Fallback:** Auto-approves static mandate from firm config YAML
- **Participates in:** Loop 6 (CEO/CIO/CRO), Loop 7 (strategy co-decision with CIO)
- Output: `MandateUpdate` event → `governance.*` topic

### CIO Agent (`src/agents/cio/cio_agent.py`)
- **LLM mode:** Claude receives `PodSummary` list + allocations + risk budgets + CEO thesis
- **Tool:** `allocate_capital(pod_id, new_pct, rationale, risk_budget_changes)`
- **Fallback:** Equal-weight (20% per pod), rebalances when any pod drifts >5%
- **Participates in:** Loop 4 (pod PM negotiation), Loop 6, Loop 7 (strategy co-decision with CEO)
- Output: `AllocationRecord` list → `CapitalAllocator.apply_allocation()`

### CEO ↔ CIO Strategy Co-Decision (Loop 7)
- CEO owns the *why* (narrative, regime thesis, objectives)
- CIO owns the *what* (capital weights, timing, rebalance plan)
- CRO joins to validate risk constraints and add hard conditions
- If no consensus after 10 iterations: hold current allocation, raise `FIRM_ALERT`
- Every co-decision transcript written to audit log

### CRO Agent (`src/agents/risk/cro_agent.py`)
- **Always rule-based. Never LLM. No exceptions.**
- Upgraded from MVP1 `RiskManager` to enforce limits across all 5 pods simultaneously
- **New checks:** firm-level VaR (sum across pods), cross-pod correlation concentration, aggregate leverage
- **Kill-switch flow:**
  1. CRO detects breach → publishes `RISK_ALERT`
  2. CRO issues `KILL_SWITCH` event (pod-level or firm-wide)
  3. Pod `PodGateway` receives it → pod halts immediately
  4. CEO + CIO notified via `governance.*` topic
- **Participates in:** Loop 5 (pod risk interrogation), Loop 6, Loop 7 (as constraint validator)

## 7. The Four New Pods

All pods follow the same 6-agent internal structure as Alpha. All have full real strategy logic.

### Pod Beta — Stat Arb / Pairs (`src/pods/templates/beta/`)
| Agent | Logic |
|---|---|
| Researcher | yfinance sector ETF flows + Alpha Vantage sector momentum |
| Signal | Rolling 60-day z-score of log price ratio per pair; pairs: (XLK/XLF), (XLE/XLV), (XLI/XLY), (XLP/XLU) |
| PM | Rule-based: enter when \|z\| > 2.0, exit when \|z\| < 0.5, stop-loss at \|z\| > 3.5 |
| Risk | Max 25% exposure per pair; monitors correlation decay; issues RiskApprovalToken |
| Execution Trader | Token validation + market/limit routing |
| Ops | Heartbeat + pair leg reconciliation (both legs always matched) |

### Pod Gamma — Global Macro (`src/pods/templates/gamma/`)
| Agent | Logic |
|---|---|
| Researcher | FRED (CPI/GDP/rates/yield curve) + Polymarket macro/Fed/election odds |
| Signal | Yield curve slope (10Y–2Y) + cross-asset 20/60-day momentum scores across SPY, TLT, GLD, UUP, EEM |
| PM | **LLM-assisted**: Claude summarises macro regime, proposes allocation weights; fallback: momentum-ranked equal-weight |
| Risk | Max 30% per asset, leverage ≤ 1.2x; issues RiskApprovalToken |
| Execution Trader | Token validation + standard routing |
| Ops | Heartbeat + FRED data staleness check |

### Pod Delta — Event-Driven (`src/pods/templates/delta/`)
| Agent | Logic |
|---|---|
| Researcher | GDELT event scoring + EDGAR 8-K freshness + Polymarket company-specific outcomes |
| Signal | GDELT event score × EDGAR 8-K freshness score × proximity-to-event decay function |
| PM | **LLM-assisted**: Claude assesses event materiality (bullish/bearish/neutral + confidence); fallback: GDELT score > 0.7 threshold |
| Risk | Max 5% per name, 1.0x leverage cap; RiskApprovalToken required; enforces 5-day auto-expiry |
| Execution Trader | Token validation + enforces 5-day position auto-expiry |
| Ops | Heartbeat + event calendar staleness check |

### Pod Epsilon — Volatility Regime (`src/pods/templates/epsilon/`)
| Agent | Logic |
|---|---|
| Researcher | CBOE VIX term structure (front/back month ratio) + FRED credit spreads + Polymarket vol-event probabilities |
| Signal | VIX term structure slope + HV/IV ratio + credit spread z-score |
| PM | Rule-based regime classifier: Low (<15) → short vol (SVXY); Normal (15–25) → neutral; High (25–35) → long vol (VXX); Extreme (>35) → max long vol + SPY hedge |
| Risk | Strict vol-of-vol limits; VXX position auto-halved at Extreme regime; issues RiskApprovalToken |
| Execution Trader | Token validation + standard routing |
| Ops | Heartbeat + VIX freshness check (stale VIX = halt; it is the core input) |

## 8. TUI New Screens

### F3 — Pod Drill-Down
Activated by pressing Enter on any pod row in F2. Shows:
- NAV, PnL, Status header
- Exposure buckets table (left) + 6-agent status with last-heartbeat times (right)
- Risk metrics panel (Vol, DD, VaR, Leverage)
- Last collaboration loop summary (participants, iterations, consensus result, duration)
- Polymarket signals feed (Gamma/Delta/Epsilon only)
- Researcher data feed (last 3 items)

### F5 — Risk Limits
Live comparison table: all 5 pods + firm row. Columns: Vol actual/limit, DD actual/limit, Leverage actual/limit, VaR status, alert indicator.
- Green: < 70% of limit
- Yellow: 70–90% of limit
- Red + blinking: > 90% of limit, `⚠` appended
- CRO auto-notified via event bus on any yellow → red transition

### F6 — Control Plane
- Capital allocation bars per pod (current %)
- Pause/Resume buttons per pod (RBAC-gated)
- [TRIGGER CEO/CIO REVIEW] button → starts Loop 7
- [FORCE REBALANCE] button → triggers CIO rebalance
- [KILL POD] and [KILL FIRM] kill-switch buttons (CRO role required)

### F9 — Building View (animated)
Textual `Timer` at 1Hz. Per-floor row shows:
- Floor number, pod name, strategy tag
- 6 agent activity dots (● active, ○ idle, ✗ halted)
- PnL (green positive, red negative)
- Risk profile: `V:{vol}% DD:{dd}% L:{lev}x`
  - Colour-coded per threshold (green/yellow/red)
  - `⚠` appended when > 90% of limit
- Capital flow arrows animate on right shaft during rebalancing
- Penthouse shows CEO/CIO/CRO + collaboration loop progress bar during deliberation
- Full-screen red flash + all floors dim on KILL_SWITCH event
- Basement shows live data feed health dots

## 10. Failure Mode Register (MVP2 additions)

| Risk | Mitigation |
|---|---|
| Polymarket API rate limit | Parquet cache + tenacity retry + fallback to empty list |
| LLM agent hallucinated allocation | Schema validation before `apply_allocation()`; CRO veto gate |
| Collaboration loop deadlock | Max iteration hard limit; no-consensus → conservative default + alert |
| CEO/CIO perpetual disagreement | Loop 7 max 10 iter; hold-current-allocation fallback |
| Cross-pod data leak via governance | Pod agents only respond via PodGateway with sanitised PodSummary |
| Gamma/Delta LLM latency in backtest | Fallback mode used in backtest by default; LLM only in live mode |
| Epsilon VIX data stale | Ops Agent heartbeat check on every tick; halt pod if VIX > 1 bar stale |
| Beta pair correlation breakdown | Signal Agent monitors correlation; PM exits position if corr < 0.5 |
| Delta position not closed at 5 days | Execution Trader enforces auto-expiry on every tick |
| AllocationRecord sum != 1.0 | CapitalAllocator.validate() rejects before apply; assertion in tests |
